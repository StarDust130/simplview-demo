# PHASE 1 — Premium Landing Page

Read every file inside the /docs folder before writing any code.

Follow the design system, coding rules, UI rules, typography rules and project vision.

This task is ONLY to build the landing page.

Do not start the dashboard.

Do not add backend code.

Do not create fake features outside the provided content.

---

## Goal

Build a world-class 2026+ AI SaaS landing page for Simplview.

The result should feel like a funded Silicon Valley startup.

The UI should immediately create a "wow" feeling.

Avoid generic SaaS templates.

Avoid dashboard-template designs.

Avoid visual clutter.

Everything should feel handcrafted.

---

## Hero Section

The hero should be centered.

Navigation at the top.

Announcement badge.

Large premium headline.

Beautiful supporting text.

Primary CTA.

Secondary CTA.

Below the hero text, place a large rounded product showcase.

This showcase should contain a beautiful embedded product video or interactive product preview.

The product preview should be the visual focus of the page.

Do NOT place the product on the right side.

It should be centered like modern SaaS websites.

---

## Background

Keep the background minimal.

Use soft blue lighting.

Soft gradients.

Subtle glow.

Very light blur.

No heavy illustrations.

No mountains.

No landscapes.

No random artwork.

No stock graphics.

The background should support the content, not compete with it.

---

## Typography

Typography is the highest priority.

Use a premium font pairing.

Large editorial headings.

Modern readable body text.

Excellent spacing.

Beautiful line height.

Strong hierarchy.

The typography alone should make the page feel expensive.

---

## Components

Cards should feel premium.

Rounded corners.

Soft shadows.

Subtle borders.

Excellent spacing.

No heavy outlines.

No clutter.

Everything should breathe.

---

## Motion

Use Framer Motion.

Very subtle animations.

Fade.

Slide.

Scale.

Soft hover animations.

Buttons feel alive.

Cards slightly lift.

Never use flashy animations.

## Code Quality

Create reusable components.

Keep files small.

Use comments to separate sections.

Avoid files over ~200 lines where practical.

Use clean folder structure.

Use TypeScript.

Use Tailwind.

Use shadcn/ui.

Follow accessibility best practices.

Follow responsive design.

Do not hardcode repeated values.

---

## Before Finishing

Review the page like a senior product designer.

Ask yourself:

- Does this feel premium?
- Does this feel modern?
- Would this fit on Awwwards?
- Would a startup founder proudly show this to investors?
- Does it feel handcrafted instead of template-based?

If the answer is no, refine it before finishing.

Only return the modified files.
